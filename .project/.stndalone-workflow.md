## Порядок публикации (из‑за зависимостей)

1. **@ui8kit/contracts** — база, без зависимостей  
2. **@ui8kit/dsl** — без @ui8kit-зависимостей  
3. **@ui8kit/lint** — без @ui8kit-зависимостей  
4. **@ui8kit/sdk** — зависит от contracts, lint, dsl  
5. **@ui8kit/generator** — зависит от sdk  



## В каждом пакете

```bash
cd packages/<name>
# Обновить version в package.json
bun run build
bun run version:patch && bun publish --access public
```

```bash
cd packages/sdk
node -e "const fs=require('fs');const j=JSON.parse(fs.readFileSync('package.json','utf8'));const[a,b,c]=j.version.split('.').map(Number);j.version=[a,b,c+1].join('.');fs.writeFileSync('package.json',JSON.stringify(j,null,2)+'\n');console.log(j.version)"
bun run build
bun publish --access public
```

**Важно:** у `contracts` и `dsl` нет `prepublishOnly`, поэтому `bun run build` нужно запускать вручную перед `npm publish`.

## В standalone-приложении

1. **Зависимости:**
   ```bash
   bun add @ui8kit/sdk @ui8kit/lint @ui8kit/generator
   ```

2. **Конфиг:** `ui8kit.config.json` в корне (из `ui8kit.config.standalone.json`).

3. **Скрипты в `package.json`:**
   ```json
   "generate": "bunx ui8kit-generate",
   "validate": "bunx ui8kit-validate",
   "lint:dsl": "bunx ui8kit-lint-dsl src"
   ```

4. **Документация:** `GUIDE.md` в каждом пакете (sdk, lint, generator).

---

## Workflow

1. **Разработка DSL** — `ui8kit-lint-dsl src`, `ui8kit-validate`  
2. **Генерация** — `bun run generate` (через `ui8kit-generate` и `ui8kit.config.json`)
